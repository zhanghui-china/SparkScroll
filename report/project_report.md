# SparkScroll项目报告书

## 1. 项目概述

### 1.1 项目名称
SparkScroll - 智能多模态内容生成与编辑平台，中文名称为”星火绘卷“。 Spark 直接致敬Nvidia DGX Spark平台，Scroll代表连环画、画卷。中文“星火绘卷”既有科技感，又有古典文学韵味。

### 1.2 项目目标
‍开发一个基于 NVIDIA GPU 加速的智能多模态内容生成与编辑平台，集成先进的大语言模型和图像编辑模型，为用户提供高效、高质量的内容创作工具。

该平台能够输入故事长文本，通过多 Agent 协作系统，全自动进行剧情压缩、分镜拆解、角色立绘设定、高保真图像渲染与后期排版，最终流式输出具有高度角色一致性的多页连环画。

### 1.3 项目背景
随着AI技术的快速发展，多模态内容生成已经成为内容创作领域的重要趋势。然而，现有的内容生成工具往往存在以下问题：
- 单模态功能限制，无法满足多场景需求
- 计算资源需求高，普通设备难以流畅运行
- 模型部署复杂，用户使用门槛高
- 生成质量参差不齐，缺乏专业性

SparkScroll 项目正是为解决这些痛点而诞生，依托 NVIDIA DGX Spark 平台的强大算力，实现高效、高质量的多模态内容生成。

### 1.4 项目要解决的实际问题
1. 降低AI模型使用门槛，让普通用户也能轻松使用先进的AI模型
2. 提供一站式多模态内容生成解决方案，涵盖文本和图像编辑
3. 利用NVIDIA GPU加速技术，提高模型推理速度和效率
4. 确保生成内容的质量和专业性，满足商业级应用需求

## 2. 作品描述

### 2.1 作品亮点

**2.1.1 平台专属优化，性能极致领先** 

本项目专为 NVIDIA DGX Spark (GB10 128G 统一内存)量身定制，采用“重型双大模型常驻显存”架构，总显存占用约 115GB，普通单卡系统完全无法运行该并发管线。只有NVIDIA DGX Spark 能实现“零 Swap”的极致秒级响应，支撑多 Agent 极速切换与流式呈现，彻底拉开与普通消费级显卡的差距。

**2.1.2 高度还原工业化出版流程** 

摒弃了简单的端到端黑盒，设计了高度还原人类工业化出版流程的 6 阶段 Agent 工作流，包括导演 Agent、编剧 Agent、立绘 Agent、出稿 Agent、后期引擎和流式呈现，实现从文本输入到最终连环画输出的全自动化流程。

**2.1.3 角色一致性与内容质量保障** 

通过立绘 Agent 预先生成角色参考图，解决了 AI 绘图“角色不连贯”的难题；采用纯代码进行后期配字，拒绝 AI 生字乱码，确保内容质量。

**2.1.4 多模态集成及良好的可扩展性** 

集成了Qwen3.5-9B等大语言模型和Qwen-Image-Edit-2511、FireRed-Image-Edit-1.1等图像编辑模型，实现文本和图像的无缝生成与编辑，提供本地模型和云端模型的任意搭配，模块化设计，支持后续集成更多先进模型。

**2.1.5 模型本地部署**

支持模型本地部署，保证数据隐私和离线使用能力。

### 2.2 作品功能

**2.2.1 核心功能** 

输入故事（如世界经典名著）长文本，通过多 Agent 协作系统，全自动进行剧情压缩、分镜拆解、角色立绘设定、高保真图像渲染与后期排版，最终流式输出具有高度角色一致性的多页连环画。

**2.2.2 功能模块说明** 

| 功能模块       | 具体描述                                                     |
| -------------- | ------------------------------------------------------------ |
| **导演 Agent** | 对原始长文本进行“降维压缩”，剔除冗余支线，精简核心角色与关键场景，提炼出紧凑的主线剧本。 |
| **编剧 Agent** | 将导演的剧本进行视觉化“镜头拆分”，规划每页 4-6 个场景，整部连环画篇幅控制在 20 页以内，输出标准化 JSON 剧本（含场景描述、留白坐标预估、对白）。 |
| **立绘 Agent** | 在正式绘制分镜前，根据导演设定的角色名单，预先绘制“核心角色三视图”与“关键物件图”，为后续分镜提供风格对齐基准。 |
| **出稿 Agent** | 接收编剧的 JSON 分镜 Prompt 与立绘 Agent 的角色参考图，生成图像时预留“构图留白”，合并输出每页的基础图像。 |
| **后期引擎**   | 通过纯代码读取编剧 JSON 中的对白文本与坐标，加载中文字体模板，将文字完美渲染至出稿 Agent 预留的空白区域。 |
| **流式呈现**   | 前端采用“流式按页呈现”策略，每生成完一页即刻推送到前端供用户翻阅，理想状态下可生成连贯的 PDF 画册。 |



## 3. 技术创新点

**3.1 重型双大模型常驻显存架构** 

采用“文本与逻辑主脑（Qwen3.5-9B）+ 视觉渲染中枢（Qwen-Image-Edit-2511）”的双大模型架构，总显存占用约 115GB，实现几万字名著“一口气”读入并保持全局剧情连贯，以及极高保真度的参考图重绘与构图留白。

**3.2 多 Agent 协作流水线** 

设计 6 阶段 Agent 工作流，高度还原人类工业化出版流程，每个 Agent 专注于特定任务，通过标准化 JSON 协议流转，实现高效协作。

**3.3 角色一致性解决方案** 

通过立绘 Agent 预先生成角色参考图，后续所有分镜均以此为基准进行风格对齐，解决了 AI 绘图中角色不连贯的行业难题。

**3.4 程序化后期配字** 

采用 Python/PIL 图像处理库进行纯代码配字，拒绝 AI 生字乱码，确保文字渲染的准确性和美观性。

**3.5 流式呈现技术** 

前端采用流式按页呈现策略，极大优化用户体验，无需等待全部内容生成即可开始翻阅。

**3.6 GPU优化策略**

采用内存优化技术，通过调整`gpu-memory-utilization`参数，在保证性能的同时降低内存使用，使模型能够在有限资源环境下运行。

**3.6 API标准化**

 设计了标准化的API接口，支持不同模型的统一调用方式，提高系统的可扩展性和可维护性。允许用户根据自己的业务定义UI界面与API服务端对接。



## 4. 使用的NVIDIA工具平台SDK开源模型

**4.1 平台适配** 

本项目专为 NVIDIA DGX Spark (GB10 128G 共享显存)平台量身定制，充分利用平台的大显存优势，实现双大模型常驻显存运行。

**4.2 工具与模型使用**

- **NVIDIA CUDA**：使用CUDA 12.9/13.0作为GPU加速基础，充分利用NVIDIA GPU的并行计算能力
- **PyTorch**：使用PyTorch 2.10.0作为深度学习框架，结合CUDA实现高效的模型推理
- **TensorRT-LLM** ：部署 Qwen3.5-9B 大语言模型，优化推理延迟，实现 128k 超长上下文处理。
- **VLLM**：使用VLLM作为本地模型的推理引擎，通过张量并行和内存优化技术，大幅提升模型推理速度
- **HuggingFace Diffusers** ：原生驱动 Qwen-Image-Edit-2511 图像编辑模型，灵活注入 Image/Prompt 条件。
- **NVIDIA GB10 硬件** ：在NVIDIA DGX Spark平台上进行模型测试和部署，提供 128G 共享显存，支撑双大模型常驻显存运行，实现“零 Swap”的极致秒级响应。
